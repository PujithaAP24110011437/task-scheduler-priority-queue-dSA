About Priority Queues & Heaps

What is a Priority Queue?

A priority queue is an abstract data type similar to a regular queue or stack data structure in which each element additionally has a "priority" associated with it. In a priority queue, an element with high priority is served before an element with low priority.

What is a Min-Heap?

A Min-Heap is a specialized tree-based data structure that satisfies the heap property: for any given node I, the value of I is less than or equal to the values of its children. This means the smallest (or highest priority) element is always at the root.

In this project, a lower priority number means the task is MORE important. Hence, we use a Min-Heap!

Why use a Heap for Task Scheduling?

Heaps provide a highly efficient way to repeatedly find and remove the smallest/largest element. This is perfect for a scheduler that constantly needs to know the "next most important task".
